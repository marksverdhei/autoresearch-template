# autoresearch

A general-purpose template for **autonomous agent-driven optimization**. Point an AI coding agent at a measurable metric, let it experiment autonomously, and wake up to a log of experiments and (hopefully) a better score.

Inspired by [Karpathy's autoresearch](https://github.com/karpathy/autoresearch), which applied this pattern to LLM pretraining. This template extracts the core principle so you can apply it to anything with a measurable outcome.

## The pattern

The idea is simple:

1. Define a **single metric** that can be computed programmatically (lower or higher is better)
2. Give an agent **target files** it's allowed to modify
3. The agent runs experiments in a loop: **modify code -> run -> measure -> keep or discard -> repeat**
4. You come back to a `results.tsv` log and a progress plot showing what worked

The agent is fully autonomous. It doesn't stop to ask questions. If an experiment improves the metric, it keeps the change. If not, it reverts. If something crashes, it logs it and moves on.

## How to use this template

1. Click **"Use this template"** on GitHub to create your own repo
2. Add your domain-specific code (the thing being optimized)
3. Fill in `program.md` with your metric, run command, and constraints
4. Fill in `autoresearch.toml` with your metric name and direction
5. Point your agent (Claude, Codex, etc.) at the repo and tell it to read `program.md`

## Project structure

```
init.md             - Step-by-step checklist to set up your project
program.md          - Agent instructions (you customize this)
autoresearch.toml   - Metric config for analysis (name + direction)
analysis.py         - Progress plotting script (reads results.tsv, writes progress.png)
pyproject.toml      - Python dependencies (just matplotlib + pandas)
results.tsv         - Experiment log (created by the agent during runs)
progress.png        - Progress plot (generated by analysis.py)
```

## Example use cases

### ML training optimization
Metric: validation loss (minimize). The agent modifies model architecture, hyperparameters, optimizer settings. Each experiment trains for a fixed time budget and reports the final metric.

### Algorithm optimization
Metric: throughput, latency, or compression ratio. The agent modifies an implementation file (sorting algorithm, encoder, search routine). Each experiment runs a benchmark and reports the score.

### LLM-as-judge scored content
Metric: judge score 0-10 (maximize). The agent modifies prompts, templates, or generation parameters. Each experiment generates outputs and runs them through an LLM judge that produces a numeric score.

### Cost / latency optimization
Metric: cost per request or p99 latency (minimize). The agent modifies API calls, caching strategies, batching logic. Each experiment runs a workload and reports the cost or timing.

### Multi-dimensional optimization
Metric: weighted aggregate score. Define a scoring function that combines multiple dimensions (quality, speed, cost) into a single number. The agent optimizes that composite score.

## The results.tsv format

The agent logs every experiment to `results.tsv` (tab-separated):

```
commit	score	status	description
a1b2c3d	0.9979	keep	baseline
b2c3d4e	0.9932	keep	increase learning rate to 0.04
c3d4e5f	1.0050	discard	switch activation function
d4e5f6g	0.0000	crash	doubled model width (OOM)
```

Columns:
- **commit**: short git hash (7 chars)
- **score**: the metric value (0.0 for crashes)
- **status**: `keep`, `discard`, or `crash`
- **description**: what the experiment tried

You can add extra columns for your domain (e.g., `memory_gb`, `latency_ms`) by extending the TSV header in `program.md`.

## Visualizing progress

```bash
uv run analysis.py
```

This reads `results.tsv` and `autoresearch.toml`, generates a progress plot, and saves it to `progress.png`.

## Requirements

- Python 3.10+
- [uv](https://docs.astral.sh/uv/) (recommended) or pip
- A coding agent (Claude Code, Codex, Cursor, etc.)
